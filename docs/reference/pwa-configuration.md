# PWA Configuration

ListMaster is a Progressive Web App powered by `vite-plugin-pwa`. The configuration lives in `vite.config.ts`.

---

## Service Worker

```ts
registerType: "autoUpdate";
```

The service worker registers with `autoUpdate` mode. When a new build is deployed, the service worker updates automatically on the next page load â€” no manual "click to update" prompt is shown. The updated service worker takes control after the page reloads.

## Asset Precaching (Workbox)

Workbox (bundled with `vite-plugin-pwa`) precaches all static assets at install time so the app works offline. The glob pattern covers:

```
js, css, html, ico, png, svg, woff2
```

On subsequent loads, assets are served from the cache first (cache-first strategy for precached assets), with the network used only for non-precached requests.

---

## GitHub Pages Base Path

```ts
// vite.config.ts
base: "/ListMaster/";
```

All generated asset URLs are prefixed with `/ListMaster/`. Without this, the browser would request `main.js` from `https://spencerking7.github.io/main.js` (the domain root) instead of `https://spencerking7.github.io/ListMaster/main.js`, resulting in 404s.

The `start_url` in the manifest also uses `/ListMaster/` for the same reason â€” the PWA must launch to the correct subpath.

## `#root` Fixed Positioning

`index.css` sets `#root` to `position: fixed; top: 0; left: 0; right: 0; bottom: 0; overflow: hidden`. This locks the app viewport in place and prevents the document from scrolling. All scrolling within the app happens inside explicitly scrollable containers (specifically the `overflow-y-auto` div in `CategoryPanel`). This is required for the iOS standalone PWA feel â€” without it, the document bounces on overscroll and the address bar intrudes.
